Data structure documentation

To know what is where

Note: Following articles decribe the data structure of the dev versions. The data structure of the pre-dev versions is explained at the end of this page.

Dev versions

This part specifies the state to the day of 5/16/2016.

Here is an ordered table of the contents of a typical Dictionary:

Init byte
Header length
General information
Number of entries
Entries

Init byte

All Dictionaries created for dev versions start with #ff to differentiate them from pre-dev Dictionaries. This is because pre-dev Dictionaries had slithly different data structure and that would otheriwise cause problems when reading the file.

Header length

Header length is a 32-bit signed integer that provides the information about the length of the general information. Specificaly, it says the length of general information part in bytes.

General information

General information holds important data about the whole Dictionary like its Author, the date of creation etc. The current ordered list is here:

Entry Data type Describtion
Name string A descriptive name that characterizes the content.
Author string The name or a nickname of the person who has written data into this Dictionary.
Identifier string A few letters long string that usually specifies the school subject of this Dictionary.
Time of creation 32-bit signed integer A Unix Timestamp that specifies the time of creation of this Dictionary. If the Dictionary was edited after that time, then the time when the main part was written is used.
Is checked? boolean A boolean value that determine whether the answer to the questions should be checked by default. This should be set to false in Dictionaries with long and/or variable answers.

Number of entries

The number of entries specifies the number of question-answer pairs in the Dictionary.

Entries

This is the last part of the Dictionary. It contains n question-answer string tuples where n is the number of entries defined above.

Pre-dev versions

The primary difference between pre-dev and dev Dictionaries is that pre-dev Dictionaries do not contain the general information.

Therefore, pre-dev Dictionaries only contain the number of entries at the beginning and then the entries themselves.
